데이터베이스 설계

 먼저 프로그램을 만들기 위해서는 데이터베이스 설계가 필요합니다. 이전 블로그 댓글을 만들 때도 간단한 데이터베이스 설계를 했었는데, 그때는 간단한 댓글 입력, 수정, 삭제만 있는 프로그램이라 설계라고 할 것도 별로 없었네요. 이번에는 하나의 완전한 프로그램을 만들기 때문에 조금 더 세밀하게 설계를 해볼까 합니다.
 
 큰 카테고리로 먼저 회원관리와 가계부 관리가 큰 카테고리네요. 그리고 저는 가계부 사용하는 사람 구성상 한국어가 아닌 다국어으로 만들어야 할 필요가 있습니다. 그래서 크게 카테고리를 지역화(Localization), 회원관리, 가계부 관리 순으로 데이터베이스를 설계해보겠습니다.
CREATE TABLE MST_LANGUAUE_TYPE(
	CODE CHAR(1) NOT NULL,
	NAME VARCHAR(5) NOT NULL,
	ISACTIVE BIT DEFAULT 1 NOT NULL,
	
	PRIMARY KEY(CODE)
);

INSERT INTO MST_LANGUAUE_TYPE (CODE, NAME, ISACTIVE) VALUES('0','kor',1);
INSERT INTO MST_LANGUAUE_TYPE (CODE, NAME, ISACTIVE) VALUES('1','eng',1);
INSERT INTO MST_LANGUAUE_TYPE (CODE, NAME, ISACTIVE) VALUES('2','jpn',1);

CREATE TABLE MST_LOCALIZATION(
	`KEY` NVARCHAR(100) NOT NULL,
	`TYPE` CHAR(1) NOT NULL,
	`VALUE` NVARCHAR(255) NOT NULL,
	ISACTIVE BIT DEFAULT 1 NOT NULL,
	
	PRIMARY KEY(`KEY`,`TYPE`),
	FOREIGN KEY (`TYPE`) REFERENCES MST_LANGUAUE_TYPE(CODE)
);

INSERT INTO MST_LOCALIZATION (`KEY`, `TYPE`, `VALUE`, ISACTIVE) VALUES(N'PrivateFinanceNote','0',N'가계부',1);
INSERT INTO MST_LOCALIZATION (`KEY`, `TYPE`, `VALUE`, ISACTIVE) VALUES(N'PrivateFinanceNote','1',N'Private finance note',1);
INSERT INTO MST_LOCALIZATION (`KEY`, `TYPE`, `VALUE`, ISACTIVE) VALUES(N'PrivateFinanceNote','2',N'`家計簿',1);
 저는 먼저 테이블을 마스터 테이블과 트랜잭션 테이블을 구분해서 작성합니다. 마스터는 시스템을 구성하는 데이터를 갖는 테이블, 트랜잭션은 프로그램상에 작성된 데이터를 갖는 테이블의 의미입니다. 지역화(Localization)는 프로그램상에서 넣는 데이터가 없기 때문에 트랜젝션 테이블이 없습니다.
 
 MST_LANGUAUE_TYPE 테이블은 언어의 구분으로 한글, 영어, 일어로 구성되어있습니다. MST_LOCALIZATION 테이블은 실제 Key 별로 프로그램 화면상에 표시될 화면입니다. 현재는 프로그램의 타이틀 영역으로 한글, 영어, 일어를 번역해서 데이터를 입력했습니다. 참고로 저는 마스터 테이블의 어두는 MST, 트랜잭션의 어두는 TSN으로 구분됩니다.
 
 여기까지 테이블 구성이 어렵지 않네요.. 다음은 회원 관리영역입니다.

블로그의 글은 모두 개인 경험과 지식으로 나온 글이기에 정확하지 않을 수 있습니다. 표준 또는 사실과 다른 글이 있으면 댓글 또는 메일 보내주세요.
