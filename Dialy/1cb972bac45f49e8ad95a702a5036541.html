Spring setting

 프로그램을 본격적으로 작성하기 전에 사전 준비로 프레임워크를 구성해야 한다.

 Java에서는 여러 가지 프레임워크와 개념이 있는데, 최근에는 Spring 프레임워크를 많이 쓴다. 각 프레임워크 특성을 다 설명하기는 힘들지만 내가 생각하기에 Spring는 DI(Dependency Injection) 의존성 주입 개념이 있는데 이게 프로그램 설계하기에 가장 편하고 구성이 잘 되어있다고 생각한다.

 DI 의존성은 객체 생성(new)을 사용자가 선언하는게 아니라 프레임워크 쪽에서 선언해서 메모리에 할당해주는 것이다. DI 의존성이라는 게 내 생각인데 사실 Singleton pattern의 상속이 안 되는 문제를 보완한게 아닐까 생각한다. 나도 무분별하게 객체 할당(new)를 좋아하지 않는 편이라 개념은 좋다라고 생각한다.

 그러나 내가 Spring 소스을 직접 분석하지는 않았지만 느낌은 Refection 개념과 디자인 패턴의 flyweight pattern일 것같다. Refection이 시스템을 동적으로 움직이게끔 하는 방식이지만 사실 이게 꽤 느리다. Singleton을 사용하는 목적이 같은 객체를 여러 번 선언하지 않음으로 성능향상을 위한 것인데, DI 의존성은 Refection을 통해 Singleton의 단점을 보완하면서 다른 쪽으로 성능을 떨어트리는....이게 뭔지... Spring이 DI의존성 개념만 있는 것은 아니지만.

그런저런 이유로 사실 나는 servlet 환경에서 내가 프레임워크 부분도 직접 설계해서 필요한부분은 DI로 사용하고 singleton으로 가능한 부분은 singleton으로 유연하게 만드는 걸 선호한다.


 그러나 이번에 내가 하는 가계부 프로젝트는 큰 사양의 시스템도 아니고 빠르게 만들고 끝내고 싶기 때문에 그냥 Spring을 쓰기로 했다. 먼저 spring을 사용하기 위해서는 maven의 pom.xml에 dependency를 아래와 같이 추가해야한다.

 나는 *.html 형태로 오면 main클래스로 *.ajax 형태로 오면 ajax 클래스를 호출하게끔 router를 만들었다. 그리고 나중에 세션 체크를 위한 공통 filter체크로 매핑시켜놓았다. 그밖에 기본 페이지인 index.html 지정과 encoding 설정을 했다.

<beans xmlns="http://www.springframework.org/schema/beans" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xmlns:mvc="http://www.springframework.org/schema/mvc" xmlns:context="http://www.springframework.org/schema/context"

      <bean class="org.springframework.http.converter.StringHttpMessageConverter">

  <bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
      <property name="prefix" value="/WEB-INF/view/"/>
      <property name="suffix" value=".jsp"/>
  </bean>
</beans>

 이건 *.ajax를 호출하면 라우팅되는 설정파일이다. ajax는 json형태로 리턴을 할 예정이기 때문에 매핑시킬 jsp 파일은 필요없다.

package Controller.Servlet;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.http.HttpSession;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller
public class Test {
  @RequestMapping(value = "/index.html")
  public String index(ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("test", "hello world!!");
    return "index";
  }
}

package Controller.Ajax;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.http.HttpSession;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller
public class Test {
  @RequestMapping(value = "/test.ajax")
  public void ajax(ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    try {
      res.getWriter().println("hello world");
    } catch (Throwable e) {
      e.printStackTrace();
    }
  }
}

 우선 나는 개발의 시작을 알리는 Hello world를 출력하기 위해 기본적인 구성만 하고 디버깅했다.

 디버깅이 되고 브라우저를 실행시키면 화면에 Hello world가 나온다. 이렇게 기본적은 프레임 워크 설정은 끝났다.
