[가계부 프로젝트 2일차] 데이터베이스 설계-1

 먼저 프로그램을 만들기 위해서는 데이터베이스 설계가 필요합니다. 이전 블로그 댓글을 만들 때도 간단한 데이터베이스 설계를 했었는데, 그때는 간단한 댓글 입력, 수정, 삭제만 있는 프로그램이라 설계라고 할 것도 별로 없었네요. 이번에는 하나의 완전한 프로그램을 만들기 때문에 조금 더 세밀하게 설계를 해볼까 합니다.  큰 카테고리로 먼저 회원관리와 가계부 관리가 큰 카테고리네요. 그리고 저는 가계부 사용하는 사람 구성상 한국어가 아닌 다국어으로 만들어야 할 필요가 있습니다. 그래서 크게 카테고리를 지역화(Localization), 회원관리, 가계부 관리 순으로 데이터베이스를 설계해보겠습니다.

해야 할 일 정리하기

  댓글 기능 추가하기 20 일차까지 해서 일단은 블로그에 댓글 기능 추가하는 것을 마무리했습니다. 아직 제가 생각하기에는 IE에서 제대로 작동 안 하는 문제 등이 있고 메일 기능 추가(댓글을 작성할 때 메일로 댓글 감사합니다(?) 메일과 리플이 달릴 때마다 리플이 달렸다는 메일 보내는 기능도 일단 생각은 했었는데 거기까지는 작업을 하지 않았습니다. 그리고 애드센스(광고)도 붙여야 합니다. 애드센스로 블로그를 하면서 돈을 많이 벌고 싶다는 의미는 아니고 앞으로 제가 해야 하면서 서버를 렌탈할 일이 생기기 때문에 그것에 대한 운영비로 사용할 예정입니다.  결론으로는 아직 남아 있는 기능에 대해서는 좀 댓글 기능을 사용해보고 사양 추가 또는 수정할 생각입니다. 연속해서 20일 포스팅을 쓰니 지루한 면도 없지 않아 있네요. 이 글을 작성하는 기능은 12월 10일입니다만 제가 연말까지 꼭 하고 싶은 작업이 있기에 서둘러 댓글 기능을 마무리한 경향도 없지 않아 있습니다.

[블로그 구축 20일차] 블로그 댓글 기능 달기 - 11(admin)

저번 포스팅까지 블로그 댓글 기능의 메인 부분은 전부 만들어졌습니다. 클라이언트(브라우저)에서 유저가 구글 아이디로 로그인하고 댓글을 작성하면 서버로 json 형태로 ajax 통신을 한다. 그리고 서버에서는 그 데이터를 받으면 데이터베이스에 저장한다. 클라이언트(브라우저)에서 유저가 해당 페이지의 댓글 리스트를 요청한다. 서버에서는 json형태로 작성하여 클라이언트로 댓글 정보를 준다. 이런 식으로 프로그램이 작성되었습니다. 아직 조그마한 버그가 눈에 띕니다만 그 부분은 포스팅 없이 수정했습니다.앞으로 조금 더 사용하면서 불편한 점이 있으면 기능을 더욱 추가할 예정입니다만 먼저 추가 기능으로는 admin기능을 추가하려고 합니다.
admin기능은 해당 댓글을 본 블로그 서비스(www.nowonbun.com)에서 관리하는 것이 아니고 서버 단에서 admin 페이지를 만들어서 관리할 예정입니다. 물론 딱히 관리페이지 없이 데이터베이스에서 직접 sql로 관리해도 상관은 없지만 불편하니 간단하게 로그인 기능 select, modify, 삭제 기능을 만들겠습니다.
