[C# スタディ - 16] static

Study / C#    作成日付 : 2019/07/10 00:19:18   修正日付 : 2019/07/10 00:20:54

こんにちは。明月です。


今日は「static」について勉強します。「static」は俗に静的キーワードと言います。「static」の種類は「staticメソッド」、「static変数」、「staticクラス」があります。


staticメソッド

今まで勉強したとおりならメソッドを使うためにはクラスを割り当てなければならないです。なぜなら、OOPの最小単位はクラスだし、メソッドはクラスの内部に存在しているからです。

でもクラスとは関係なく、計算だけするメソッドが必要な時もあります。例えば、パラメタで「int」タイプデータを２つを受け取って、パラメタの数を足す計算だけするメソッドがあると思いましょう。そしてそのメソッドを使うためにクラスを割り当てる(new)ことはメモリが惜しい感じがしますね。

その時に「static」メソッドに実装すればクラスを割り当てずにメソッドを使うことができます。

アクセス修飾子 static 返却データタイプ メソッド名(パラメタ...)
{
  return 返却値
}

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace staticExample
{
  class StaticEx
  {
    /// <summary>
    /// static宣言Sumメソッド
    /// </summary>
    /// <param name="a">パラメタ1</param>
    /// <param name="b">パラメタ2</param>
    /// <returns>パラメタ1+パラメタ2</returns>
    public static int Sum(int a, int b)
    {
      return a + b;
    }
  }
  class Program
  {
    static void Main(string[] args)
    {
      int a = 10;
      int b = 20;

      //割当が必要ではなくSumメソッドを使える。
      int c = StaticEx.Sum(a, b);

      Console.WriteLine(c);

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
  }
}

上の例を見ると「new」(割り当て)をしなくてメソッドを使います。そうすれば、staticメソッドからクラス内部のメンバ変数を使えるかと疑問になりますね。

上をみるとエラーが発生しますね。なぜなら「static」メソッドの場合はクラスを割り当てて使うものではなく静的領域なので、動的なクラスの部分に直接に参照ができないです。

でも、メソッドの中でクラスを割り当てたら使えます。

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace staticExample
{
  class StaticEx
  {
    private int data;
    public int Data
    {
      get
      {
        return data;
      }
      set
      {
        data = value;
      }
    }
    /// <summary>
    /// static宣言Sumメソッド
    /// </summary>
    /// <param name="a">パラメタ1</param>
    /// <param name="b">パラメタ2</param>
    /// <returns>パラメタ1+パラメタ2</returns>
    public static int Sum(int a, int b)
    {
      StaticEx test = new StaticEx();
      test.Data = a + b;
      return test.Data;
    }
  }
  class Program
  {
    static void Main(string[] args)
    {
      int a = 10;
      int b = 20;

      //割当が必要ではなくSumメソッドを使える。
      int c = StaticEx.Sum(a, b);

      Console.WriteLine(c);

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
  }
}


static変数

static変数は消されない変数だといいます。すなわち、クラスの変数はクラスが割り当てる時に変数の値が生成してクラスが解除する時に変数の値が無くなります。

でも、プロジェクトの中で唯一に宣言するし、解除が行わない変数も必要だと思いますね。その時に使う変数を「static」変数と言います。

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace staticExample
{
  class StaticEx
  {
    static int sumData = 0;
    
    /// <summary>
    /// static宣言Sumメソッド
    /// </summary>
    /// <param name="a">パラメタ1</param>
    /// <returns>累積パラメタ1</returns>
    public static int Sum(int a)
    {
      return sumData += a; ;
    }
  }
  class Program
  {
    static void Main(string[] args)
    {
      Console.WriteLine(StaticEx.Sum(10));
      Console.WriteLine(StaticEx.Sum(20));
      Console.WriteLine(StaticEx.Sum(30));

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
  }
}


staticクラス

「staticクラス」はクラスの構成メンバがすべて構成が「static」になります。そしてstaticクラスの場合はコンストラクタをもってられません。

なぜなら、「staticクラス」は割り当て(new)られないので、コンストラクタが存在できません。

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace staticExample
{
  static class StaticEx
  {
    //定数
    public const int YEAR_HEISEI_28= 2016;

    //static変数
    private static int sumData = 0;
    /// <summary>
    /// 変数sumdataのプロパティ
    /// </summary>
    public static int SumData
    {
      get
      {
        return sumData;
      }
      set
      {
        sumData = value;
      }
    }
    /// <summary>
    /// static宣言Sumメソッド
    /// </summary>
    /// <param name="a">パラメタ1</param>
    /// <returns>累積パラメタ1</returns>
    public static int Sum(int a)
    {
      return SumData += a; ;
    }
  }
  class Program
  {
    static void Main(string[] args)
    {
      Console.WriteLine(StaticEx.Sum(StaticEx.YEAR_HEISEI_28));
      Console.WriteLine(StaticEx.Sum(1));
      Console.WriteLine(StaticEx.SumData);

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
  }
}
