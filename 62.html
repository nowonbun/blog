[Java] ジェネリック(Generic type)

Study / Java    作成日付 : 2019/09/06 22:30:49   修正日付 : 2020/01/17 07:41:46

こんにちは。明月です。


このページはジェネリック(Generic type)に関して調べてみました。


クラスの基本構成はメンバー変数、関数になります。

public class Example {
  public static void main(String[] args) {
    Stack obj = new Stack();
    for (int i = 0; i < 10; i++) {
      // int型で入力
      obj.push(i);
    }
    while(true) {
      // Object型でデールのでこのデータが「int」型かはしらないね。
      Object data = obj.pop();
      if(data == null) {
        break;
      }
      System.out.println(data);
    }
  }
}

class Stack {
  // メモリポインタのため、あるクラス
  class Node {
    Object data;
    Node link;
  }

  private Node pointer = null;

  // データをクラスに入力する。
  public void push(Object data) {
    Node node = new Node();
    node.data = data;
    node.link = pointer;
    this.pointer = node;
  }
  // 一番、最後に「push」されたことが出る構造
  public Object pop() {
    if (pointer == null) {
      return null;
    }
    Node node = this.pointer;
    this.pointer = node.link;
    return node.data;
  }
}

上の例はStackアルゴリズムを実装しました。Stackアルゴリズムは「push」、「pop」の関数でなっているアルゴリズムです。

「push」の順番で「pop」のより一番最後に入力されたデータが一番最初に出る構造です。(FILO構造)


上のアルゴリズムでパラメータは「Object」型になっていますね。なので「String」や「int」や「float」、「double」のデータタイプでも入力します。

でも、「pop」にして使うようには「Object」型なので、ユーザによってデータタイプを知らないですね。逆にデータキャストをしなければならない構造です。

そうすれば「Object」ではなく、データタイプをクラス内部ではなく、外部で決める方法がないかというとそれをジェネリック(Generic type)で解決できます。

public class Example {
  public static void main(String[] args) {
    //ジェネリックを「Integer」に設定しているから「Stack」のクラス内部の「T」と書いてることがすべて「Integer」タイプに切り替えると思えば楽。
    Stack<Integer> obj = new Stack<>();
    for (int i = 0; i < 10; i++) {
      obj.push(i);
    }
    while(true) {
      //　返却データタイプが「Integer」になっているからキャストが必要ない。
      Integer data = obj.pop();
      if(data == null) {
        break;
      }
      System.out.println(data * 10);
    }
  }
}
// ジェネリック(Generic type)で内部ではデータタイプを決めてない。
class Stack<T> {
  class Node {
    // データタイプはジェネリックだ。
    T data;
    Node link;
  }

  private Node pointer = null;

  // パラメータのデータタイプはジェネリックだ。
  public void push(T data) {
    Node node = new Node();
    node.data = data;
    node.link = pointer;
    this.pointer = node;
  }

  //返却データタイプはジェネリックだ。
  public T pop() {
    if (pointer == null) {
      return null;
    }
    Node node = this.pointer;
    this.pointer = node.link;
    return node.data;
  }
}

Eclipseのコードアシスタントを確認しても「pop」関数の返却タイプが「Integer」になっているし「push」のパラメータも「Integer」タイプになっていますね。

我々がよく使う関数「List」タイプでもすべてジェネリック(Generic type)になっていますね。

ジェネリック(Generic type)は単純なキーワードですが、プログラムのデータ正誤性のため、様々のところで重要なキーワードですね。
