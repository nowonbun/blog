[Java スタディ - 36] Spring web frameworkでControllerを使用する方法

Study / Java    作成日付 : 2019/09/27 20:39:09   修正日付 : 2019/09/27 20:47:44

こんにちは。明月です。


このページでは「Spring web framework」で「Controller」を使用する方法に関して調べてみました。


私が前述で「Spring web framework」でウェブサービスを構築する方法に関して投稿しました。

link - [Java スタディ - 35] EclipseでSpring web frameworkを利用してウェブサービスプロジェクトを立ち上がる方法


「Spring web framework」では様々なアトリビュートがあります。その中で「RequestMapper」と「RequestParam」が大事なのでその２つに関して確認しましょう。

RequestMapper

コントローラを見ると「RequestMapper」のアトリビュートを利用して「request」ルータを管理しています。

package controller;

import javax.servlet.http.*;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller
public class Helloworld {
  //リスト形式のマッピング
  @RequestMapping(value = { "/", "/index.html", "/helloworld.html" })
  public String index(ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", "Helloworld");
    return "index";
  }
}

「RequestMapping」は特定な「url」だけではなく、同時に様々な「url」を複数でマッピングができます。


package controller;

import javax.servlet.http.*;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller
public class Helloworld {
  // urlが「i」で始まるアドレス
  @RequestMapping(value = "i*.html")
  public String index(ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", "index");
    return "index";
  }
  // urlが「h」で始まるアドレス
  @RequestMapping(value = "h*.html")
  public String helloworld(ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", "Helloworld");
    return "index";
  }
}

「RequestMapping」は指定するurlだけではなく、正規表現式でマッピング設定ができます。


package controller;

import javax.servlet.http.*;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;
import org.springframework.web.bind.annotation.*;

@Controller
public class Helloworld {
  @RequestMapping(value = "{id}.html")
  public String index(@PathVariable String id, ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", id + "index");
    return "index";
  }
}

「RequestMapping」のデータを関数の内部で使うこともできます。上の例みたいに「{id}」のキーを指定して関数のパラメータ「@ParhVariable」で取得ができます。

結果は「index」.html」の場合は「index」、「helloworld.html」の場合は「helloworld」が出力しますね。


package controller;

import javax.servlet.http.*;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;
import org.springframework.web.bind.annotation.*;

@Controller
@RequestMapping(value = "/test")
public class Helloworld {
  @RequestMapping(value = "{id}.html")
  public String index(@PathVariable String id, ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", id + " page");
    return "index";
  }
}

要請されたRequestのディレクトリ別で分けてクラスを呼出すこともできます。


package controller;

import javax.servlet.http.*;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;
import org.springframework.web.bind.annotation.*;

@Controller
@RequestMapping(value = "/test")
public class Helloworld {
  //　GETメソッドタイプのRequest
  @RequestMapping(value = "{id}.html", method = RequestMethod.GET)
  public String index1(@PathVariable String id, ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", id + " page");
    return "index";
  }
  // POSTメソッドタイプのRequest
  @RequestMapping(value = "{id}.html", method = RequestMethod.POST)
  public String index2(@PathVariable String id, ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", id + " page");
    return "index";
  }
}


RequestParam

RequestParamはブラウザから受取るパラメタの値です。関数のパラメタで「HttpServletRequest」と「HttpServletResponse」を受取るので、このパラメタがなくてもデータは受取るのができますね。

では関数別のパラメタで呼出す関数を決めることやアノテーションを利用してデータ取得するのが綺麗にデータ処理が可能なので、利用する方が良いですね。

package controller;

import javax.servlet.http.*;
import org.springframework.web.bind.annotation.*;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;

@Controller
public class Helloworld {
  @RequestMapping(value = "index.html", method = RequestMethod.GET)
  public String index(@RequestParam("param") String param, ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", "parameter - " + param);
    return "index";
  }
}

上の例はパラメタから「param」のキーでブラウザからサーバにデータを送っています。結果は関数で「param」を受け取ってデータを「view」に渡します。


package controller;

import javax.servlet.http.*;
import org.springframework.web.bind.annotation.*;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;

@Controller
public class Helloworld {
  // index.htmlでパラメタ(param)がある場合に。
  @RequestMapping(value = "index.html", params = { "param" }, method = RequestMethod.GET)
  public String index1(@RequestParam("param") String param, ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", "parameter - " + param);
    return "index";
  }
  // index.htmlでパラメタがない場合に。
  @RequestMapping(value = "index.html", method = RequestMethod.GET)
  public String index2(ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", "Hello world");
    return "index";
  }
}

上の例はパラメタがあることとないことで呼出す関数が違いますね。


package controller;

import javax.servlet.http.*;
import org.springframework.web.bind.annotation.*;
import org.springframework.stereotype.Controller;
import org.springframework.ui.ModelMap;

@Controller
public class Helloworld {
  // 基本データタイプではなく、クラスタイプでも自動に変数名を合わせてデータを入れる。
  @RequestMapping(value = "index.html", params = { "param" }, method = RequestMethod.GET)
  public String index(@ModelAttribute Node node, ModelMap modelmap, HttpSession session, HttpServletRequest req, HttpServletResponse res) {
    modelmap.addAttribute("Data", "parameter - " + node.getParam());
    return "index";
  }
}

// Node クラスの中で param変数に パラメタ paramの値が入る。
class Node {
  private String param;
  public String getParam() {
    return param;
  }
  public void setParam(String param) {
    this.param = param;
  }
}

パラメタを基本変数ではなく、クラスタイプでも受取るのができます。例えば、Html上でPOSTタイプでinputデータ値が多い場合に全ての変数名をパラメタで受取ることではなく、「bean」クラスとしてデータを受取るのができます。


その以外に「header」の要請別、タイプ別でも呼出すメソッドを分岐できます。このページでは内容が長すぎるので次のControllerでajax要請する時のjson変換するところで合わせて調べましょう。
