[C# スタディ - 23] 例外処理(try〜catch,throwそしてfinally)

Study / C#    作成日付 : 2019/07/15 02:25:26   修正日付 : 2019/07/15 02:26:17

こんにちは。明月です。


今日は例外処理について勉強します。例外処理はキーワード名をみればエラーを解決するキーワードみたいですが実はそうではありません。大きくみると全域の分岐というの意味が近いと思われます。

この全域の分岐はエラー処理するところでよく使われるので例外処理と呼ばれるかも知れません。


try〜catch、throw

例外処理はプロセス処理順「try」ブロックの内で「Exception」クラスタイプを「throw」から投げれば「catch」で受け取ります。

try
{
  throw new Exception();//Exceptionをcatchに投げる。 
}
catch(Exception e)
{
  //Exceptionを受け取る。
}

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace exceptionEx
{
  class Program
  {
    static void Main(string[] args)
    {
      try
      {
        //tryのブロック
        String data = "ExceptionEx";
        //catchにExceptionを投げる
        throw new Exception("ERROR WORLD");
        Console.WriteLine(data);
      }
      //throwからExceptionを受け取りました。
      catch (Exception e)
      {
        Console.WriteLine(e.Message);
      }

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
  }
}

上の例をみると「if」文と差がなさそうですね。でも「try〜catch,throw」は「try」ブロックの内で「throw」を発生することを「catch」で受け取るものなので「ｔｒｙ」のブロックのメソッド内で「throw」を投げても「catch」で受け取ります。

﻿using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace exceptionEx
{
  class Program
  {
    static void Main(string[] args)
    {
      try
      {
        //tryのブロック
        String data = "ExceptionEx";
        Process();
        Console.WriteLine(data);
      }
      //throwからExceptionを受け取りました。
      catch (Exception e)
      {
        Console.WriteLine(e.Message);
      }

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
    //メソッド内で「throw」を投げる。
    static void Process()
    {
      //catchにExceptionを投げる
      throw new Exception("ERROR Method WORLD");
    }
  }
}

そして継承関係でも「try〜catch、throw」ができます。すなわち親クラスで「Exception」を「throw」で投げたら派生クラスの「try〜catch」で受け取ることができます。

﻿using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace exceptionEx
{
  class ParentClass
  {
    //親クラスから派生クラスに「throw」を投げるメソッド
    public virtual void Process()
    {
      //catchにExceptionを投げる
      throw new Exception("ERROR Abstract WORLD");
    }
  }
  class ChildClass : ParentClass
  {
    public override void Process()
    {
      try
      {
        //tryのブロック
        String data = "ExceptionEx";
        base.Process();
        Console.WriteLine(data);
      }
      //throwからExceptionを受け取りました。
      catch (Exception e)
      {
        Console.WriteLine(e.Message);
      }
    }
  }
  class Program
  {
    static void Main(string[] args)
    {
      //「ChildClass」を割り当て
      ChildClass child = new ChildClass();
      child.Process();

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
  }
}

ここまで確認すると確かに一般分岐と差がありますね。一般分岐の場合は条件によって処理プロセスが変わりますが、「try〜catch、throw」の場合は「try」ブロックの内で「throw」を投げると「catch」にで受け取ることで処理になります。


finally

「try〜catch」の場合は処理中で投げる形態だから「throw」の次のレベルソースが実行しないです。そうことになると必ず実行すべきなソースレベルがある場合に論理的なエラーがなる可能性があります。

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace exceptionEx
{
  class ExceptionExClass
  {
    public void Process()
    {
      int i = 0;
      while (i < 10)
      {
        //無限ループになる。
        try
        {
          //「method」メソッドで「Exception」が発生
          method();
          //それで「i」の値が増減しない
          i++;
        }
        catch (Exception e)
        {
          Console.WriteLine(e.Message);
        }
      }
    }
    public void method()
    {
      throw new Exception("ERROR");
    }
  }
  class Program
  {
    static void Main(string[] args)
    {
      //「ExceptionExClass」を割り当て
      ExceptionExClass child = new ExceptionExClass();
      child.Process();

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
  }
}

上の例を見ると「method」で「Exception」が発生すれば「i++」は実行しないので無限ループになります。(実は例みたいに極端的に実装しない。)

それでエラーが発生(throw)しても無限ループにならないように「finally」キーワードを使います。

﻿using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace exceptionEx
{
  class ExceptionExClass
  {
    public void Process()
    {
      int i = 0;
      while (i < 10)
      {
        //無限ループになる。
        try
        {
          //「method」メソッドで「Exception」が発生
          method();
        }
        catch (Exception e)
        {
          Console.WriteLine(e.Message);
        }
        finally
        {
          //「Exception」が発生しても必ず実行する「finally」
          i++;
        }
      }
    }
    public void method()
    {
      throw new Exception("ERROR");
    }
  }
  class Program
  {
    static void Main(string[] args)
    {
      //「ExceptionExClass」を割り当て
      ExceptionExClass child = new ExceptionExClass();
      child.Process();

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
  }
}

「finally」の場合は「try」のブロックの内で返却「return」しても必ず処理するブロックです。なのでプログラムリソースを解除するところでよく使いますね。

﻿using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace exceptionEx
{
  class ExceptionExClass
  {
    public void Process()
    {
      //無限ループになる。
      try
      {
        //返却する。
        return;
      }
      catch (Exception e)
      {
        Console.WriteLine(e.Message);
      }
      finally
      {
        //返却しても必ず実行する「finally」
        Console.WriteLine("finally!!!!!!");
      }
    }
  }
  class Program
  {
    static void Main(string[] args)
    {
      //「ExceptionExClass」を割り当て
      ExceptionExClass child = new ExceptionExClass();
      child.Process();

      Console.WriteLine("Press Any Key...");
      Console.ReadLine();
    }
  }
}
